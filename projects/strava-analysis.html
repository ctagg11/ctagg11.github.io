Analyzing My Strava Running Data

January 2025 Python Data Analysis Visualization Strava API

Overview

In this project, I analyzed my personal running data from Strava to uncover patterns in my training habits, performance trends, and the relationship between various metrics like distance, pace, and elevation gain.

The goal was to answer questions like: Am I getting faster? What days do I run most? How does elevation affect my pace? This analysis combines data extraction from the Strava API, data cleaning, exploratory analysis, and visualization.

Key Findings

1. Training Volume Over Time

I wanted to see how my weekly mileage has evolved. The chart below shows a clear upward trend, with some obvious taper periods before races.

Python 
# Weekly mileage
weekly = df.groupby(df['date'].dt.to_period('W'))['distance_km'].sum()

plt.figure(figsize=(12, 5))
plt.plot(weekly.index.to_timestamp(), weekly.values, linewidth=2)
plt.fill_between(weekly.index.to_timestamp(), weekly.values, alpha=0.3)
plt.title('Weekly Running Volume', fontsize=16, fontweight='bold')
plt.xlabel('Week')
plt.ylabel('Distance (km)')
plt.grid(True, alpha=0.3)
plt.tight_layout()
plt.show()

2. Pace Distribution by Day of Week

Interestingly, my Tuesday runs are consistently faster - probably because those are my track workout days!

Python 
# Order days properly
day_order = ['Monday', 'Tuesday', 'Wednesday', 'Thursday', 'Friday', 'Saturday', 'Sunday']
df['day_of_week'] = pd.Categorical(df['day_of_week'], categories=day_order, ordered=True)

plt.figure(figsize=(12, 6))
sns.boxplot(data=df, x='day_of_week', y='avg_pace_min_km')
plt.title('Pace Distribution by Day of Week', fontsize=16, fontweight='bold')
plt.xlabel('Day')
plt.ylabel('Average Pace (min/km)')
plt.xticks(rotation=45)
plt.tight_layout()
plt.show()

3. Elevation vs. Pace Correlation

As expected, there's a positive correlation between elevation gain and pace - hillier routes slow me down. The relationship is roughly 0.1 min/km slower per 100m of elevation gain.

Python 
from scipy.stats import pearsonr

# Calculate correlation
corr, p_value = pearsonr(df['elevation_gain_m'], df['avg_pace_min_km'])
print(f"Correlation: {corr:.3f}, p-value: {p_value:.4f}")

plt.figure(figsize=(10, 6))
plt.scatter(df['elevation_gain_m'], df['avg_pace_min_km'], alpha=0.5)
plt.xlabel('Elevation Gain (m)', fontsize=12)
plt.ylabel('Average Pace (min/km)', fontsize=12)
plt.title(f'Elevation vs Pace (r={corr:.2f})', fontsize=14, fontweight='bold')

# Add trend line
z = np.polyfit(df['elevation_gain_m'], df['avg_pace_min_km'], 1)
p = np.poly1d(z)
plt.plot(df['elevation_gain_m'], p(df['elevation_gain_m']), "r--", alpha=0.8, linewidth=2)
plt.grid(True, alpha=0.3)
plt.tight_layout()
plt.show()

Conclusions

  • Training Progress: Weekly mileage increased by ~40% over the analysis period
  • Pace Improvement: Average pace improved from 5.8 to 5.5 min/km
  • Consistency: Tuesday and Saturday are my most consistent running days
  • Terrain Impact: Elevation significantly impacts pace - important for race planning

This analysis provided valuable insights into my training patterns and helped identify areas for improvement. Next steps include analyzing heart rate data and comparing performance across different weather conditions.
